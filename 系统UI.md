# 研究生科研工作台系统 UI 方案

## 1. 设计目标

本系统的 UI 不应做成传统后台管理台，也不应做成纯聊天页。建议定义为：

**研究工作台 + 资料空间 + 生成式助手**

设计关键词：

- 轻科技
- 学术感
- 留白
- 玻璃雾面
- 柔和蓝青紫渐变
- 卡片化工作流

参考你提供的截图，建议保留以下优点：

- 白底大留白，页面显得轻
- 卡片边缘柔和，阅读不压迫
- 蓝紫与青色作为功能强调色
- 左侧导航 + 中央主工作区 + 右侧辅助信息区

但需要避免直接照搬社区/资源平台结构，因为你的产品核心是“研究工作流”，不是内容分发。

## 2. 总体视觉方向

## 2.1 风格定位

建议采用：

**云雾玻璃学术风**

视觉特征：

- 背景使用极淡的蓝白渐变和雾状光斑
- 卡片为半透明白面板，轻阴影，边框极浅
- 重点按钮使用青蓝到靛蓝渐变
- 页面图标走线性风格，避免厚重商务感
- 数据、引用、标签统一做成轻胶囊形态

## 2.2 字体建议

- 中文正文：`MiSans` 或 `HarmonyOS Sans SC`
- 中文标题：`Alibaba PuHuiTi 3.0`
- 英文数字：`IBM Plex Sans`
- 代码/引用：`JetBrains Mono`

不建议使用默认系统字体堆栈作为主视觉字体，否则页面会过于普通。

## 3. 信息架构

## 3.1 一级导航

- 首页
- 学术调研
- 知识库
- 网页搜索
- AI 服务
- 研究计划
- 个人中心

## 3.2 推荐导航结构

顶部导航：

- Logo
- 一级导航
- 全局搜索
- 消息中心
- 用户头像菜单

左侧辅助导航：

- 当前模块二级菜单
- 常用筛选项

中央主区域：

右侧信息区：

- 统计数据
- 最近任务 / 引用源 / 历史记录

## 4. 核心页面方案

## 4.1 首页工作台

### 页面目标

让用户一进入系统就知道今天该做什么、资料在哪、最近研究进行到哪里。

### 布局

- 顶部 Hero 区
  - 今日欢迎语
  - 当前研究项目
  - 快捷入口：开始检索 / 上传资料 / 新建计划
- 中部左栏
  - 最近搜索
  - 收藏文献
  - 最近生成结果
- 中部右栏
  - 今日任务
  - 番茄钟
  - 打卡状态
- 底部
  - 项目空间卡片
  - 学术趋势或推荐资料

### 关键交互

- 支持从首页直接发起研究问题
- 支持拖动调整卡片排序
- 支持快速回到上次会话

## 4.2 学术调研页

### 页面目标

围绕“研究问题”而不是“关键词”设计。

### 布局

- 顶部搜索区
  - 研究问题输入框
  - 高级筛选：年份、来源、作者、领域、类型
- 左侧筛选栏
  - 时间范围
  - 论文类型
  - 研究方法
  - 是否已收藏/已入库
- 中间结果区
  - 文献卡片列表
  - 支持列表视图和关系图视图
- 右侧洞察区
  - AI 快速总结
  - 关键主题
  - 建议追问
  - 一键加入知识库

### 文献卡片字段

- 标题
- 作者
- 发表年份
- 来源
- 摘要
- 标签
- 引用次数或热度
- 操作按钮：查看详情、总结、收藏、入库、加入项目

### 增强能力

- 关系图模式：显示作者、文献、主题之间关系
- 证据模式：展示句级引用片段
- 时间轴模式：查看主题演进

## 4.3 知识库页

### 页面目标

让“资料归档”和“资料再利用”都顺手。

### 布局

- 左侧：知识库树
  - 知识库
  - 文件夹
  - 标签
- 中间：文档列表
  - 文件名
  - 来源
  - 上传时间
  - 状态
  - 标签
- 右侧：文档预览与问答
  - 摘要
  - 命中片段
  - 引用定位
  - 问答面板

### 关键交互

- 拖拽上传
- 批量删除与批量打标签
- 文档详情抽屉
- 支持“从搜索结果直接入库”
- 支持“从 AI 输出反向存为知识条目”

## 4.4 网页搜索页

### 页面目标

服务“资料检索 + 内容沉淀”，而不是普通搜索引擎页。

### 布局

- 搜索框
- 平台标签：全部 / 网页 / 小红书 / 抖音 / 医学站点 / 实验工具站
- 中间结果流
- 右侧 Markdown 摘要预览

### 关键交互

- 支持将多条结果合并为一个 Markdown 摘要
- 支持保存为“网页资料卡”
- 支持发送到 AI 助手继续加工

## 4.5 AI 服务页

### 页面目标

不是做成单一聊天框，而是做成“助手工作台”。

### 结构

- 左侧：助手类型
  - 提示词优化
  - Bio 医学助手
  - 知识库问答
  - 文献简报
  - 网页总结
- 中间：会话流
- 右侧：来源资料、可插入变量、模板、历史版本

### 关键交互

- 选择来源资料后再发问
- 每条回答显示引用来源
- 回答可一键保存成笔记、任务、报告草稿

## 4.6 研究计划页

### 页面目标

把研究行动管理起来，而不是只停留在“搜到了什么”。

### 页面结构

- 顶部：今日专注时长、完成任务数、连续打卡天数
- 中部左：任务清单
- 中部中：日历
- 中部右：番茄钟
- 底部：周报视图

### 核心交互

- 搜索结果可直接转成任务
- AI 输出可直接生成计划项
- 番茄钟结束自动提醒记录成果

## 4.7 个人中心

### 页面目标

既是个人名片，也是研究资产陈列页。

### 结构

- 左侧：头像、简介、方向、绑定信息
- 中间：我的项目、我的收藏、我的产出
- 右侧：研究画像
  - 近期关键词
  - 高频主题
  - 活跃时间

## 6. 交互与动效建议

## 6.1 页面动效

- 页面首次加载使用 240ms 渐入 + 轻微上移
- 卡片分组以 40ms 阶梯式出现
- 右侧洞察栏采用横向滑入

## 6.2 反馈动效

- 收藏使用轻弹跳
- 保存成功用右上轻 Toast，不要重遮罩
- 任务完成用渐变描边闪动一次

## 6.3 搜索体验

- 输入研究问题后先显示骨架屏
- 搜索完成后顶部显示“已找到 X 条结果”
- 支持筛选条件吸顶

## 7. 响应式策略

## 7.1 桌面端

- 1440px 以上采用三栏布局
- 1280px 左右可压缩为两栏 + 抽屉右栏

## 7.2 平板端

- 左侧筛选折叠为抽屉
- 右侧洞察区收纳到 Tab

## 7.3 移动端

- 首页仅保留关键卡片
- 学术调研改成单列流式卡片
- 知识库问答与文档预览分步进行
- 番茄钟置顶，日历下沉

## 9. 视觉落地建议

### 设计重点一

首页一定要强，必须做出“研究工作台入口感”，不能像传统控制台。

### 设计重点二

搜索页不要只做结果列表，必须做“结果 + AI 洞察 + 保存动作”的三联布局。

### 设计重点三

知识库页的价值不在上传，而在“引用回溯 + 问答 + 二次产出”。

### 设计重点四

AI 页不要做成聊天产品翻版，应强化来源、模板、可复用输出。

## 10. 总结

这套 UI 建议以“轻、净、专业、可沉浸”为目标：

- 外观参考 ModelScope 的轻科技感
- 结构参考研究工作台而非社区页
- 交互强调搜索、保存、生成、计划四个动作闭环
- 视觉上做到足够现代，但不花哨到影响阅读

如果后续要继续落地前端页面，我建议优先做：

1. 首页工作台
2. 学术调研页
3. 知识库页
4. AI 服务页

这四页决定了产品是否“像一个系统”。
